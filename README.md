# 🌸 Weather App

This project was built to practice working with real APIs and asynchronous
logic in a front-end–only environment using vanilla JavaScript.

The focus was on handling external data, managing application state in the
browser, and building a clean, responsive UI without frameworks.

## Live Demo
https://a41900.github.io/weather-app/

## Purpose
The goal of this project was to better understand how to fetch, process, and
display real-time data from external APIs, while dealing with common front-end
constraints such as rate limits, errors, and client-side state.

## What I worked on
- Fetching weather data from an external API
- Calculating local time based on city timezones
- Caching API responses using localStorage
- Handling failed API requests and edge cases
- Managing dynamic background images via the Unsplash API
- Building a responsive UI with smooth background transitions

## API Keys & Security Notes
This is a client-side–only application deployed on GitHub Pages.
Because the application runs entirely in the browser, API keys are exposed by
design. The keys used belong to free-tier APIs and are intended for learning and
portfolio demonstration purposes only.
In a production environment, API requests should be handled by a backend proxy
to avoid exposing sensitive credentials.
